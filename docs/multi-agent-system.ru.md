# Многоагентная система и A2A протокол

[English version](./multi-agent-system.md)

`fagent` поддерживает оркестрацию множества агентов через A2A (Agent-to-Agent) протокол, обеспечивая динамическое создание агентов, межагентную коммуникацию и параллельное выполнение задач.

## Обзор

Многоагентная система расширяет возможности fagent за пределы одноагентных и субагентных паттернов:

- **Субагенты**: Временные фоновые воркеры с ограниченным набором инструментов
- **Именованные агенты**: Постоянные агенты с полной конфигурацией
- **A2A протокол**: Стандартизированная коммуникация между агентами

## Типы агентов

### Главный агент

Основной оркестратор, координирующий других агентов.

### Субагенты

Исполнители фоновых задач, создаваемые через `spawn()`:

```python
spawn(
    task="Исследовать тему X",
    label="Исследование",
    model="anthropic/claude-haiku-4"  # Опциональный выбор модели
)
```

**Особенности:**
- Ограниченный доступ к инструментам (нет spawn, нет message)
- Автоматическая очистка после завершения
- Выбор модели для каждого субагента

### Именованные агенты

Постоянные агенты, создаваемые через `create_agent()`:

```python
create_agent(
    name="researcher",
    model="anthropic/claude-sonnet-4",
    temperature=0.7,
    disable_vector_memory=True
)
```

## Инструменты A2A протокола

### create_agent

Создание нового постоянного агента с кастомной конфигурацией.

**Параметры:**
- `name`: Идентификатор агента
- `model`: Используемая модель
- `temperature`: Опциональное переопределение температуры
- `max_tokens`: Опциональный лимит токенов
- `disable_vector_memory`: Отключить векторную память
- `disable_shadow_context`: Отключить теневой контекст

### send_to_agent

Отправка сообщений другим агентам через message bus.

**Параметры:**
- `agent_id`: ID целевого агента
- `message`: Содержимое сообщения

### list_agents

Список всех зарегистрированных агентов с их статусом.

## Конфигурация

### Конфигурация для каждого агента

```toml
[agents.defaults]
model = "anthropic/claude-opus-4-5"
temperature = 0.1

[agents.agents.researcher]
model = "anthropic/claude-sonnet-4"
temperature = 0.7
disable_vector_memory = true
disable_shadow_context = false

[agents.agents.coder]
model = "anthropic/claude-opus-4-5"
max_tokens = 16384
```

### CLI команды

```bash
# Создать агента
fagent agent create researcher --model anthropic/claude-sonnet-4

# Список агентов
fagent agent list

# Показать конфигурацию
fagent agent config researcher

# Удалить агента
fagent agent delete researcher
```

## Паттерны оркестрации

### Последовательное делегирование

```
1. Главный агент получает сложную задачу
2. Создаёт специализированных агентов
3. Делегирует подзадачи последовательно
4. Агрегирует результаты
```

### Параллельное выполнение

```
1. Определяет независимые подзадачи
2. Создаёт несколько субагентов
3. Выполняет параллельно
4. Собирает результаты
```

### Иерархическая координация

```
Главный агент (Оркестратор)
├── Агент исследования (сбор данных)
├── Агент анализа (обработка данных)
└── Агент отчётов (генерация вывода)
```

## Реализация

### AgentRegistry

Управление жизненным циклом агентов:

```python
from fagent.agent.registry import AgentRegistry

registry = AgentRegistry(bus=message_bus, workspace=workspace)
agent_id = registry.register_agent(name, model, config)
```

### Интеграция с Message Bus

Агенты общаются через `MessageBus`:

```python
await registry.send_message_to_agent(
    agent_id="abc123",
    content="Проанализируй эти данные",
    sender_id="main"
)
```

## Best Practices

1. **Выбор модели**: Используйте быстрые модели (haiku) для простых задач
2. **Управление памятью**: Отключайте неиспользуемые системы памяти для stateless агентов
3. **Границы задач**: Держите задачи агентов сфокусированными и чётко определёнными
4. **Лимиты ресурсов**: Мониторьте количество одновременных агентов
5. **Обработка ошибок**: Проверяйте статус агентов и обрабатывайте сбои

## Пример: Команда исследователей

```python
# Создать специализированных агентов
create_agent(name="data_collector", model="haiku")
create_agent(name="analyst", model="sonnet")

# Делегировать задачи
send_to_agent(data_collector, "Собери рыночные данные за Q1 2026")
# Ждём ответа
send_to_agent(analyst, "Проанализируй эти данные: {data}")

# Синтезируем результаты
```

## Ограничения

- Агенты используют общее рабочее пространство
- Нет межпространственной коммуникации
- Message bus находится в памяти (не персистентный)
- Субагенты имеют ограниченный доступ к инструментам
